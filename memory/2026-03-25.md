## 📚 今日学习：Webhook 实现要点

### 核心概念
- Webhook 是 HTTP 回调，用于事件驱动架构
- 接收方提供 URL，发送方在事件发生时 POST 数据
- 需要验证签名确保请求来源可信

### 实现要点
1. **端点设计**: POST /webhook/:source
2. **签名验证**: HMAC-SHA256 验证 payload
3. **幂等性**: 通过 event_id 去重
4. **异步处理**: 接收后快速响应 200，后台处理
5. **重试机制**: 发送方失败时重试，接收方需支持幂等

### 应用场景
- GitHub/GitLab push 事件
- 支付成功通知（Stripe/支付宝）
- CI/CD 流水线触发
- 监控告警推送
